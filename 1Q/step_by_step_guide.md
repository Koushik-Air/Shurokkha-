## ✅ WHAT TO DO — Step by Step

### Step 1: Set Up Firebase (MUST DO FIRST)

Nothing works without Firebase. Do this before anything else.

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Create a project (or use your existing one)
3. Register Android app with package name: `com.shurokkha.shurokkha`
4. Download `google-services.json` → place it in `android/app/`
5. Register iOS app with the same bundle ID
6. Download `GoogleService-Info.plist` → place it in `ios/Runner/`
7. Enable these Firebase services:
   - **Authentication** → enable Email/Password and Phone providers
   - **Cloud Firestore** → create database in test mode
   - **Cloud Storage** → enable storage

### Step 2: Set Up Google Maps API Key

1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Enable **Maps SDK for Android**, **Maps SDK for iOS**, and **Places API**
3. Create an API key (restrict it to your package name for production)
4. Open `android/app/src/main/AndroidManifest.xml`
5. Replace `YOUR_GOOGLE_MAPS_API_KEY_HERE` with your real key

### Step 3: Verify the Current Fixes Compile

Before doing anything else, confirm the codebase is clean:

```bash
cd /home/koushik/Desktop/Projects/Thesis
flutter pub get
flutter gen-l10n
flutter analyze
```

You should see: **"No issues found!"**

If you see errors, do NOT proceed to the improvement prompt. Fix them first.

### Step 4: Run the Improvement Prompt in Gemini Flash 3.5

1. Open Gemini Flash 3.5 (AI Studio, Antigravity, or Gemini app)
2. Open `1Q/gemini_improve_prompt.md`
3. Copy **only** the content inside the ` ``` ` code fence (after "THE PROMPT")
4. Paste it as your first message
5. Gemini will reply with a **plan** — read it carefully
6. Reply `continue phase 1` to get Phase 1 code
7. **Apply Phase 1 files to your project**
8. Run `flutter analyze` to verify zero errors
9. **Only then** reply `continue phase 2`
10. Repeat for all 6 phases

### Step 5: After Each Phase — Verify

After applying each phase's code:

```bash
# If new dependencies were added:
flutter pub get

# If ARB localization files changed:
flutter gen-l10n

# If new Hive models were added:
dart run build_runner build --delete-conflicting-outputs

# Always check for errors:
flutter analyze
```

### Step 6: Test on a Real Device

```bash
flutter run
```

Test the following flow:
1. App opens as Calculator → type `9876=` → app unlocks
2. Sign up with email + phone + password
3. Add at least 1 emergency contact
4. Long-press the SOS button → countdown appears → cancel it
5. Open Settings → change SOS message → save
6. Open Map → verify directions button launches Google Maps
7. Shake the phone → verify countdown triggers

## 🚫 WHAT NOT TO DO

### ❌ Do NOT paste the entire .md file into Gemini

The `.md` file contains instructions for YOU (the developer). Only paste the
content inside the ` ``` ` code fence. If you paste the full markdown including
headers and instructions, Gemini will get confused.

### ❌ Do NOT skip phases or do them out of order

Each phase builds on the previous one. Phase 3 (volume trigger) modifies
`MainActivity.kt`, and Phase 4 (launcher swap) modifies it again. If you skip
Phase 3, Phase 4 will be missing the volume-button code.

**Correct order: 1 → 2 → 3 → 4 → 5 → 6**

### ❌ Do NOT apply code without running `flutter analyze` first

If Gemini generates code with a typo or wrong API (e.g., `onPressed` instead
of `onTap` on a `ListTile`), you must catch it before moving to the next phase.
Run `flutter analyze` after every single phase.

### ❌ Do NOT say "do it all" or "generate everything at once"

Gemini Flash 3.5 has a **65,536 token output limit**. If you ask for all phases
at once, the response will be truncated mid-file and you'll get broken code.
Always go **one phase at a time**.

### ❌ Do NOT modify generated localization files manually

These files are auto-generated by `flutter gen-l10n`:
- `lib/core/localization/l10n/app_localizations.dart`
- `lib/core/localization/l10n/app_localizations_en.dart`
- `lib/core/localization/l10n/app_localizations_bn.dart`

If you edit them, your changes will be **overwritten** the next time you run
`flutter gen-l10n`. Always edit the `.arb` files instead.

### ❌ Do NOT modify `safe_place_model.g.dart`

This file is generated by `build_runner`. If you edit it manually, it will break
or be overwritten. To change the model, edit `safe_place_model.dart` and re-run:

```bash
dart run build_runner build --delete-conflicting-outputs
```

### ❌ Do NOT commit `google-services.json` or API keys to Git

These contain your private Firebase and Google Cloud credentials. Add them to
`.gitignore`:

```
android/app/google-services.json
ios/Runner/GoogleService-Info.plist
```

### ❌ Do NOT test SOS features on the emulator

The following features require a **real Android/iOS device**:
- SMS sending (telephony plugin)
- Shake detection (accelerometer)
- Volume-button trigger (hardware keys)
- Background audio recording (microphone)
- GPS location tracking

The emulator will silently fail or return mock data for all of these.

### ❌ Do NOT use the fix prompt again — it's already applied

The `gemini_fix_prompt.md` has already been executed. All its changes are in
your codebase. Running it again will generate duplicate code or overwrite
improvements from the improvement prompt.

### ❌ Do NOT ignore Gemini's "plan" response

When you paste a prompt, Gemini replies with a **plan first** (no code). This
is your chance to catch problems before code is generated. Read it. If something
looks wrong, tell Gemini to adjust the plan before saying `continue phase 1`.

### ❌ Do NOT forget to handle the `telephony` deprecation warning

The `telephony: ^0.2.0` package is discontinued. It works for now but may
break on future Android versions. Plan to replace it with a custom platform
channel or a maintained fork. For your thesis, it's fine as-is.

## 📁 File Ownership Rules

| File Type | Who edits it | How to regenerate |
|---|---|---|
| `.arb` files | You / Gemini | — |
| `app_localizations*.dart` | NEVER edit | `flutter gen-l10n` |
| `*.g.dart` files | NEVER edit | `dart run build_runner build --delete-conflicting-outputs` |
| `pubspec.yaml` | You / Gemini | `flutter pub get` after changes |
| `AndroidManifest.xml` | You / Gemini | — |
| `MainActivity.kt` | You / Gemini | — |
| All other `.dart` files | You / Gemini | — |
